Connect to Live Data
Enter the connection URL provided by your CIO Office contact — updates sync automatically
Step 1 — Create the Google Sheet
  1. Go to sheets.new and create a blank spreadsheet
  2. Name it: CIO Portfolio Management — FY2026
  3. Copy the Sheet ID from the URL:
    https://docs.google.com/spreadsheets/d/SHEET_ID_HERE/edit
  4. Keep this tab open — you'll come back to it
Step 2 — Deploy the Apps Script
  1. In your Sheet: Extensions → Apps Script
  2. Delete the default Code.gs content
  3. Paste in the code from Code.gs (downloaded below)
  4. Replace YOUR_GOOGLE_SHEET_ID_HERE with your actual Sheet ID
  5. Run setupSheet() once to create all tabs
  6. Click Deploy → New Deployment
  7. Type: Web App · Execute as: Me · Who has access: Anyone
  8. Copy the Web App URL
Step 3 — Connect this Tracker
Paste the deployment URL from Step 2 here
How it works once connected
What syncs automatically
  • Every project add / edit / delete → writes to Projects tab
  • Every demand request → writes to Demands tab
  • Every weekly status update → writes to Weekly tab
  • Every roadmap change → writes to Roadmap tab
  • Every stakeholder edit → writes to Stakeholders tab
  • Changelog entries → appended to Changelog tab
What this enables
  • You + Jane + Sujith all see the same live data
  • BU leads submit demand requests directly — you see it instantly
  • The Sheet is always a clean Excel-ready backup
  • Generate Word / PPT reports directly from the Sheet
  • Share the HTML file — anyone with the GAS URL connected sees live data
  • Refresh page → always current, no manual export needed
Important: Each person who opens the HTML file needs to enter the GAS URL once in Step 3 above — it saves in their browser. After that, every change they make syncs to the shared Sheet automatically. The HTML file itself does not need to be re-shared after data changes.
⚠ Data safety: Entering a URL here and clicking "Save & Test" will never delete your local data. "Pull from Sheets" only overwrites local data when Sheets contains data. "Push to Sheets" only writes to the Sheet. Your browser's localStorage copy is always preserved unless you explicitly clear your browser data.
🛟 Data Diagnostic & Recovery
What each button does:
🔍 Run Diagnostic — scans every possible storage key and shows exactly what's saved
🛟 Restore from Browser Storage — loads data from the most recent saved key and re-renders all views
🗂 Scan All Storage Keys — lists every key in localStorage so you can see what's there
